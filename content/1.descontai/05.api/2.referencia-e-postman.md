# Referencia da API

Referencia rapida de **todos os endpoints** do backend, agrupados por recurso, mais a
colecao Postman pronta para importar. Para arquitetura e fluxos internos, veja o
[Guia de manutencao](./1.guia-de-manutencao).

## Base e autenticacao

- **Base URL (producao):** `https://descontai-api-wtkrc.ondigitalocean.app`
- **Prefixo:** todos os recursos ficam sob `/api/v1` (o `/health` fica na raiz).
- **Autenticacao:** JWT Bearer. Faca `POST /api/v1/auth/login`, use o `token` retornado
  no header `Authorization: Bearer <token>`. O token expira em **30 minutos**; renove com
  `POST /api/v1/auth/refresh` usando o `refreshToken`.

Legenda de acesso: 🔓 publico · 🔑 autenticado · 👑 ADMIN · 🏢 EMPRESA

## Postman

Colecao completa (96 requests) com autenticacao Bearer automatica — o request de
**Login** ja salva o `token`/`refreshToken` nas variaveis via script de testes.

- [Colecao (descontai.postman_collection.json)](/descontai/postman/descontai.postman_collection.json)
- [Environment de producao](/descontai/postman/descontai-prod.postman_environment.json)
- [Environment local](/descontai/postman/descontai-local.postman_environment.json)

**Como usar:** importe os 3 arquivos → selecione o environment → rode `Auth > Login` →
os demais requests herdam o Bearer token automaticamente.

## Auth · `/api/v1/auth`

| Metodo | Rota | Acesso | Descricao |
|---|---|---|---|
| POST | `/auth/register` | 🔓 | Cria conta (email, senha, nome, cidadeId, cidade) |
| POST | `/auth/login` | 🔓 | Autentica e retorna `token` + `refreshToken` |
| POST | `/auth/refresh` | 🔓 | Renova o par de tokens |
| POST | `/auth/logout` | 🔑 | Invalida o refresh token |
| POST | `/auth/esqueci-senha` | 🔓 | Solicita e-mail de redefinicao |
| POST | `/auth/redefinir-senha` | 🔓 | Redefine a senha com o token do e-mail |

::callout{type="warning" title="Cuidado ao testar"}
Todas as rotas 👑/🏢 negam acesso com **HTTP 403** quando o usuario nao tem a role
necessaria. Os endpoints que enviam notificacao push, criam pagamentos reais ou registram
contas devem ser testados em ambiente de sandbox/staging, nao em producao.
::
