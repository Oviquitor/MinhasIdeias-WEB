Nesta aula você conhece as duas ferramentas que sustentam todo o app: **Dart** (a linguagem em que a
gente escreve) e **Flutter** (o framework que desenha a tela). E a ideia central que, uma vez
entendida, faz o Flutter inteiro fazer sentido: **tudo é um widget**.

## Dart: a linguagem

**Dart** é a linguagem de programação criada pelo Google que o Flutter usa. Se você já viu qualquer
linguagem parecida com C, Java ou JavaScript, vai se sentir em casa: tem variáveis, `if`, `for`,
funções, classes. Nada de assustador.

```dart
// uma variável e uma função em Dart
String saudacao(String nome) {
  return 'Olá, $nome!';   // $nome insere a variável no texto (interpolação)
}

void main() {
  final mensagem = saudacao('Ana'); // 'final' = valor que não muda depois
  print(mensagem);                  // imprime: Olá, Ana!
}
```

Três coisas do Dart que você vai ver o tempo todo no Descontai:

::fields
::field{name="Null safety (segurança contra nulo)"}
Uma variável `String` **não pode** ser nula; se puder, você escreve `String?` (com `?`). Isso obriga
você a **pensar no caso vazio** e evita o erro mais comum de apps: o "null" que quebra a tela. Repare
que no `SessionCache` o token é `String? token` — pode não existir (usuário deslogado).
::
::field{name="async / await (código que espera)"}
Chamar a API **demora**. Métodos que esperam são marcados com `async` e você usa `await` para
aguardar o resultado sem travar a tela. Ex.: `await _dio.post(...)` — "espere a API responder e siga".
Isso vai aparecer em **todo** serviço do app.
::
::field{name="Classes e construtores"}
Tudo em Dart é organizado em **classes**. Um `HomeController`, um `LoginResponse`, um `SessionCache`
— todos são classes. É o mesmo conceito de orientação a objetos que você viu na API (em Java).
::
::

::callout{type="tip" title="Não precisa dominar Dart antes de começar"}
Você aprende Dart **usando** — igual a andar de bicicleta. Cada aula introduz só o que precisa. Se
bater dúvida numa sintaxe (`?`, `=>`, `..`), volte aqui: são poucos símbolos e eles se repetem o
tempo todo. O importante agora é entender a **ideia** do Flutter, que vem a seguir.
::

## Flutter: o framework

**Flutter** é o kit de ferramentas (framework) que pega o seu código Dart e o transforma num app que
roda em Android e iOS. Ele traz duas coisas principais: um **motor de renderização** (que desenha
cada pixel na tela, rápido) e uma **biblioteca gigante de componentes prontos** (botões, listas,
campos de texto, animações).

O diferencial do Flutter: ele **desenha tudo por conta própria**. Ele não usa os botões do Android
nem os do iOS — ele **pinta** os seus. Por isso o app fica **idêntico** nas duas plataformas e você
tem controle total do visual.

## A grande ideia: tudo é um widget

Aqui está o conceito que destrava o Flutter. No Flutter, **absolutamente tudo na tela é um widget**:
um texto é um widget (`Text`), um botão é um widget, um espaçamento é um widget (`Padding`), uma
coluna que empilha coisas é um widget (`Column`), até a tela inteira é um widget.

::callout{type="info" title="Widget, na analogia do Lego"}
Pense em **peças de Lego**. Cada peça é pequena e faz uma coisa só. Você **encaixa** peças pequenas
para formar peças maiores, e peças maiores para formar a construção inteira. Um **widget** é uma
peça de Lego da interface: você combina widgets pequenos (um texto, um ícone) dentro de widgets
maiores (um card) dentro de widgets ainda maiores (uma tela). **A tela inteira é uma montagem de
widgets encaixados.**
::

Um exemplo simplificado de um card de cupom, para você sentir o encaixe:

```dart
Card(                                   // um widget "cartão"
  child: Column(                        // dentro dele, uma coluna que empilha...
    children: [
      Text('Pizzaria do Zé'),           // ...um texto (widget)
      Text('30% de desconto'),          // ...outro texto (widget)
      ElevatedButton(                   // ...e um botão (widget)
        onPressed: () {},
        child: Text('Usar cupom'),      // o texto DENTRO do botão também é widget
      ),
    ],
  ),
)
```

Repare: widgets **dentro** de widgets **dentro** de widgets. Isso forma a **árvore de widgets**.

## A árvore de widgets

Como cada widget contém outros, o app inteiro é uma **árvore**: lá em cima está o app (`MaterialApp`),
que contém a tela, que contém uma coluna, que contém cards, que contêm textos e botões. Quando o
Flutter precisa desenhar, ele **percorre essa árvore** de cima a baixo, montando a tela.

```text
MaterialApp
 └─ Scaffold (estrutura de uma tela: appbar, corpo, etc.)
     ├─ AppBar
     └─ Column
         ├─ Text('Olá, Ana')
         └─ Card
             └─ Text('30% off')
```

Você vai ver essa aninhamento (uma coisa dentro da outra, com bastante indentação) o tempo todo no
Descontai. No começo assusta pela quantidade de parênteses; depois vira leitura natural — é só a
árvore desenhada em código.

## Hot reload: o superpoder do Flutter

Aqui está o que torna desenvolver em Flutter **viciante**. Enquanto o app roda, você muda o código
(a cor de um botão, o texto de um título), salva, e **em menos de um segundo** a mudança aparece na
tela **sem reiniciar o app** — mantendo até onde você estava. Isso é o **hot reload**.

::callout{type="tip" title="Por que hot reload muda o jogo"}
No desenvolvimento nativo tradicional, cada mudancinha visual pode exigir recompilar e reinstalar o
app — minutos de espera a cada ajuste. Com o hot reload, o ciclo "muda → vê" é **instantâneo**. Você
experimenta, erra, corrige e aprende numa velocidade absurda. É a melhor amiga do dev mobile — e a
razão de eu insistir que você **rode o app** enquanto estuda.
::

## Material 3: o kit visual pronto

O Descontai usa o **Material Design 3** (ou "Material 3" / "Material You"), o sistema de design do
Google. Na prática, isso te dá widgets prontos e bonitos — `Scaffold`, `AppBar`, `Card`,
`ElevatedButton` — já com comportamento e aparência padronizados. Você parte de uma base pronta e
customiza (cores, fontes) em vez de desenhar tudo do zero. Vamos ver isso na Aula 5, quando falarmos
de tema.

## Resumindo

- **Dart** é a linguagem (parecida com Java/JS): variáveis, classes, `async/await`, e **null safety**
  (`String?` avisa que pode ser nulo).
- **Flutter** é o framework que transforma o Dart num app Android **e** iOS, **desenhando** cada
  pixel por conta própria (visual idêntico nas duas plataformas).
- A ideia central: **tudo é um widget**. Você **encaixa** widgets pequenos em widgets maiores, como
  peças de Lego, formando a **árvore de widgets**.
- O **hot reload** aplica mudanças na tela em menos de um segundo, sem reiniciar o app — o ciclo de
  aprendizado mais rápido que existe.
- O Descontai usa **Material 3**, que dá componentes prontos e bonitos como ponto de partida.

::callout{type="tip" title="Fixando"}
Olhe qualquer tela de um app no seu celular e tente "ver os widgets": *esse título é um `Text`, esse
botão é um widget, isso aqui empilhado é uma `Column`, esse cartão é um `Card`...*. Quando você
começa a enxergar telas como **árvores de widgets encaixados**, você começou a pensar em Flutter.
::
