Você já sabe que tudo é widget e que existem stateless e stateful. Agora vamos **montar telas de
verdade**: como empilhar, alinhar e espaçar widgets (layout), e como manter o visual **consistente**
em todo o app (tema). São duas habilidades que andam juntas.

## Scaffold: o esqueleto de uma tela

Quase toda tela do Descontai começa com um **`Scaffold`** — o widget que dá a **estrutura básica** de
uma página Material: um lugar para a barra do topo (`appBar`), o corpo (`body`) e uma barra
inferior/botão flutuante, se precisar.

```dart
Scaffold(
  appBar: AppBar(title: const Text('Meus cupons')), // barra do topo
  body: Center(                                       // corpo da tela
    child: Text('Conteúdo aqui'),
  ),
)
```

::callout{type="info" title="Scaffold = andaime"}
"Scaffold" quer dizer **andaime**. É a armação onde você pendura o resto da tela: topo, corpo, rodapé.
No Descontai, a `BasePage` (a tela principal com as abas) é um Scaffold com uma AppBar de cidade em
cima e uma barra de navegação flutuante embaixo. Você quase sempre parte de um Scaffold.
::

## Empilhar e alinhar: Column e Row

Os dois widgets de layout mais usados no Flutter:

::fields
::field{name="Column" type="empilha na vertical ↓"}
Coloca os filhos **um embaixo do outro**. É o que monta a maioria das telas (título, depois um card,
depois um botão). No card de cupom da Aula 4, a `Column` empilhava o nome e o desconto.
::
::field{name="Row" type="alinha na horizontal →"}
Coloca os filhos **lado a lado**. Ótimo para um ícone ao lado de um texto, ou um preço à esquerda e
um botão à direita.
::
::

```dart
Column(                          // empilha na vertical
  crossAxisAlignment: CrossAxisAlignment.start, // alinha à esquerda
  children: [
    const Text('Pizzaria do Zé'),
    Row(                         // lado a lado
      children: const [
        Icon(Icons.local_offer),
        SizedBox(width: 8),      // um espacinho horizontal
        Text('30% de desconto'),
      ],
    ),
  ],
)
```

## As ferramentas de espaço: Padding, SizedBox e Expanded

Três widgets que você vai usar sem parar para deixar a tela respirar:

::fields
::field{name="Padding" type="espaço INTERNO"}
Afasta o conteúdo das bordas. `Padding(padding: EdgeInsets.all(16), child: ...)` dá 16 pixels de
folga em volta. É o "respiro" que separa o texto da borda da tela.
::
::field{name="SizedBox" type="espaço fixo"}
Uma caixa de tamanho definido. Muito usada como **espaçador**: `SizedBox(height: 12)` cria 12 pixels
de espaço vertical entre dois widgets.
::
::field{name="Expanded" type="ocupa o que sobra"}
Dentro de uma Row/Column, faz o filho **esticar** para preencher o espaço livre. Dois `Expanded` lado
a lado dividem a largura meio a meio. É como resolver "quero que isso ocupe o resto da linha".
::
::

::callout{type="warning" title="O erro de layout mais comum (e como ler o erro)"}
Cedo ou tarde você vai ver um erro amarelo-e-preto tipo *"RenderFlex overflowed"*. Ele quase sempre
significa: **"tentei colocar mais coisa do que cabe na tela"** (um texto longo demais numa Row, por
exemplo). A solução costuma ser envolver o filho num `Expanded` ou `Flexible`, ou permitir rolagem
com um `SingleChildScrollView`. Não entre em pânico — esse erro é didático: ele te diz **onde** não
coube.
::

## Tema: o visual consistente em um lugar só

Imagine ter que escrever "cor laranja `#FF6B00`, fonte 16, negrito" em **cada** botão do app. Se um
dia a marca mudar de cor, você caçaria isso em centenas de arquivos. A solução é o **tema**: você
define as cores, fontes e estilos **num lugar central**, e todas as telas herdam dali.

No Descontai, isso é organizado em três arquivos (repare como cada um tem uma responsabilidade única):

| Arquivo | Responsabilidade |
|---|---|
| `lib/core/theme/app_theme.dart` | O tema Material do app (cores base, fontes, estilos de botão) |
| `lib/shared/theme/app_colors.dart` | A **paleta** de cores da marca, com nomes (ex.: `AppColors.primary`) |
| `lib/shared/theme/app_sizes.dart` | Tamanhos e **cálculos responsivos** (espaçamentos, fontes que se adaptam à tela) |

O tema é ligado no `main.dart`, no `MaterialApp`:

```dart
MaterialApp.router(
  title: 'Descontai',
  theme: AppTheme.light,     // ← o tema definido em app_theme.dart
  routerConfig: AppRouter.router,
)
```

A partir daí, um widget pega cores e estilos do tema em vez de valores soltos:

```dart
// em vez de escrever a cor na mão em cada lugar:
Container(color: const Color(0xFFFF6B00)) // ❌ espalha a cor pelo código

// você usa a paleta central:
Container(color: AppColors.primary)        // ✅ um lugar só define a cor
```

::callout{type="tip" title="Por que centralizar cores e tamanhos vale ouro"}
Centralizar o visual é o mesmo princípio de "não se repetir" que guia todo bom código. Muda a cor da
marca? **Um** arquivo. Precisa escurecer todos os títulos? **Um** lugar. Além de consistência (todas
as telas ficam iguais de verdade), você ganha manutenção barata. É por isso que `app_colors.dart` e
`app_sizes.dart` existem — eles são a **fonte única da verdade** do visual.
::

## Responsividade: telas de tamanhos diferentes

Um celular pequeno e um tablet têm tamanhos bem diferentes. Se você fixar tudo em pixels, o layout
quebra num aparelho e sobra espaço em outro. Por isso o Descontai tem o `app_sizes.dart`, com
**cálculos responsivos**: espaçamentos e fontes que se adaptam ao tamanho da tela, em vez de números
mágicos fixos.

::callout{type="info" title="A ideia da responsividade"}
Em vez de "esse título tem 24 pixels", você pensa "esse título é proporcional à largura da tela".
Assim ele fica bom no celular pequeno **e** no grande. O `app_sizes.dart` concentra esses cálculos
para que as telas não precisem repetir a conta — elas só pedem "o tamanho de título", e o arquivo
resolve conforme o aparelho.
::

## Resumindo

- **`Scaffold`** é o esqueleto de uma tela Material: `appBar` (topo), `body` (corpo) e barras
  inferiores. Quase toda tela parte dele.
- **`Column`** empilha na vertical; **`Row`** alinha na horizontal. Juntos, montam a maioria dos
  layouts.
- **`Padding`** (espaço interno), **`SizedBox`** (espaço fixo/espaçador) e **`Expanded`** (ocupa o
  espaço livre) são as ferramentas de espaço do dia a dia.
- O erro *"overflowed"* significa "não coube" — resolva com `Expanded`/`Flexible` ou rolagem.
- O **tema** centraliza o visual: `app_theme.dart` (tema Material), `app_colors.dart` (paleta) e
  `app_sizes.dart` (tamanhos responsivos). Muda num lugar, reflete em todo o app.

::callout{type="tip" title="Fixando"}
Pegue uma tela do Descontai na cabeça e "monte ela em widgets": *é um `Scaffold`, com uma `AppBar` em
cima; o `body` é uma `Column` com um título, um `SizedBox` de respiro, e uma lista de `ProdutoCard`;
as cores vêm de `AppColors`.* Se você consegue descrever a tela como layout + tema, você já pensa como
quem constrói interfaces.
::
