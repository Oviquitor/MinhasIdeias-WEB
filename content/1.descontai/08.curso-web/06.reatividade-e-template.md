Esta é a aula mais importante do Vue. **Reatividade** é o que faz a tela se atualizar sozinha quando os
dados mudam — sem você mexer em elementos na mão. E as **diretivas** são os comandos que ligam os dados
ao HTML. Vamos aprender tudo com a tela real de **listagem de cupons**.

## ref: um dado reativo

Um **`ref`** é uma "caixa" que guarda um valor **reativo**: quando o valor muda, tudo na tela que
depende dele se atualiza automaticamente. Você cria com `ref(valorInicial)` e acessa/muda o conteúdo
com `.value`:

```ts
const busca = ref('')        // uma caixa reativa começando vazia
busca.value = 'pizza'        // muda o valor → a tela reage
console.log(busca.value)     // lê o valor
```

Na tela de cupons real, veja o estado sendo declarado com `ref`:

```ts
const cupons = ref<ProdutoResponse[]>([])  // a lista (começa vazia)
const loading = ref(true)                   // está carregando?
const erro = ref('')                        // mensagem de erro
const busca = ref('')                       // texto do campo de busca
```

::callout{type="info" title="Por que .value?"}
No `<script>`, você mexe no valor com `.value` (`loading.value = false`). Já no `<template>`, o Vue
"desembrulha" a caixa sozinho — você escreve só `{{ busca }}`, sem `.value`. Confunde no começo: **com
`.value` no script, sem `.value` no template**. É o preço de o Vue saber exatamente **quais** dados são
reativos para atualizar só o necessário.
::

## computed: valor derivado (calculado)

Um **`computed`** é um valor que **deriva** de outros dados reativos — ele **recalcula sozinho** quando
qualquer um deles muda. É perfeito para "a lista filtrada pela busca". Olha o real:

```ts
// Recalcula automaticamente sempre que 'cupons' OU 'busca' mudarem.
const filtrados = computed(() => {
  const q = busca.value.trim().toLowerCase()
  if (!q) return cupons.value                    // busca vazia → todos
  return cupons.value.filter(
    (c) => c.produto.toLowerCase().includes(q) ||
           (c.codigo || '').toLowerCase().includes(q),
  )
})
```

::callout{type="tip" title="computed x função comum"}
Você poderia fazer uma função `filtrar()` e chamá-la toda hora. Mas o `computed` é melhor: ele
**recalcula sozinho** quando `busca` ou `cupons` mudam (o Vue rastreia as dependências), e **guarda o
resultado** (cache) enquanto nada muda. Regra: se um valor **deriva** de outros dados reativos, use
`computed` — não uma variável comum que você teria que lembrar de atualizar. É o mesmo "derive, não
guarde" que vimos no app.
::

## As diretivas: ligando dados ao HTML

**Diretivas** são atributos especiais (começam com `v-` ou `:` ou `@`) que dão comportamento reativo ao
HTML. As cinco que você usa o tempo todo:

### v-if / v-else-if / v-else — mostra ou esconde

Renderiza um elemento **só se** a condição for verdadeira. A tela de cupons tem os três estados (Aula
14) com isso:

```vue
<div v-if="erro" class="erro">{{ erro }}</div>          <!-- só se houver erro -->
<div v-else-if="loading">Carregando…</div>               <!-- senão, se carregando -->
<DcEmpty v-else-if="!filtrados.length" ... />            <!-- senão, se lista vazia -->
<div v-else>...grade de cupons...</div>                  <!-- senão, os dados -->
```

### v-for — repete para cada item

Renderiza um elemento **para cada item** de uma lista. É como monta a grade de cards:

```vue
<div v-for="c in filtrados" :key="c.id" class="dc-card cupom">
  <div class="title">{{ c.produto }}</div>
</div>
```

::callout{type="warning" title="Sempre use :key no v-for"}
Repare no `:key="c.id"`. O Vue usa a `key` (um identificador único) para saber **qual** item é qual
quando a lista muda — assim ele atualiza só o que mudou, em vez de redesenhar tudo. Esquecer o `:key`
causa bugs sutis (itens embaralhados, estado no card errado). **Todo `v-for` precisa de um `:key`
único** — decore isso.
::

### v-bind (`:`) — liga um atributo a um valor

Você já viu na Aula 5: `:prop="variavel"` passa o **valor** de uma expressão para um atributo. Serve
para qualquer atributo HTML — inclusive estilo dinâmico:

```vue
<img :src="c.fotoUrl" alt="" />                     <!-- o src vem da variável -->
<span :class="isAtivo(c) ? 'on' : 'off'">...</span>  <!-- classe condicional -->
```

### v-on (`@`) — reage a eventos

Executa código quando algo acontece (clique, envio de formulário). É a "eletricidade" (Aula 2):

```vue
<button @click="confirmDelete = c">Remover</button>   <!-- ao clicar, guarda o cupom -->
<form @submit.prevent="onSubmit">...</form>            <!-- ao enviar, chama onSubmit -->
```

O `.prevent` (no login real) é um **modificador**: evita o comportamento padrão do navegador (que
recarregaria a página ao enviar o form). Pequeno, mas essencial numa SPA.

### v-model — ligação de mão dupla

O mais mágico: **`v-model`** liga um campo de input a um `ref` nos **dois sentidos**. O usuário digita →
o `ref` muda; o `ref` muda → o campo mostra. Na busca de cupons:

```vue
<input v-model="busca" placeholder="Buscar cupom…" />
```

::callout{type="info" title="v-model é o que faz a busca 'funcionar sozinha'"}
Pensa no fluxo: o usuário digita "pizza" → `v-model` atualiza o `ref` `busca` → o `computed`
`filtrados` (que depende de `busca`) recalcula → o `v-for` que renderiza `filtrados` redesenha a grade
com só os resultados. **Você não escreveu nenhum "quando digitar, filtre e redesenhe"** — a reatividade
encadeou tudo sozinha. Digitar → filtrar → mostrar, de graça. Esse encadeamento é o superpoder do Vue.
::

## O ciclo de vida: onMounted

Uma última peça: **quando** carregar os dados? Você quer buscar os cupons **assim que a tela aparece**.
Para isso existe o `onMounted` — um "gancho de ciclo de vida" que roda quando o componente é montado na
tela:

```ts
async function load() {
  loading.value = true
  try {
    cupons.value = await api.listCupons()   // busca na API
  } catch (e: any) {
    erro.value = e?.message || 'Erro ao carregar os cupons.'
  } finally {
    loading.value = false
  }
}
onMounted(load)   // ao montar a tela, executa load()
```

Repare no padrão: **liga o loading → busca → em erro, guarda a mensagem → sempre desliga o loading**
(`finally`). É o mesmo ciclo do `HomeController` do app — a mecânica de carregar dados é universal.

## Resumindo

- **`ref`** guarda um dado **reativo**: mudou o `.value` → a tela reage. (`.value` no script, sem
  `.value` no template.)
- **`computed`** é um valor **derivado** que recalcula sozinho quando suas dependências mudam (e faz
  cache) — como a lista `filtrados`.
- Diretivas ligam dados ao HTML: **`v-if/v-else`** (mostra/esconde), **`v-for`** (repete, sempre com
  **`:key`**), **`v-bind`/`:`** (liga atributo a valor), **`v-on`/`@`** (reage a eventos, com
  modificadores como `.prevent`) e **`v-model`** (ligação de mão dupla em inputs).
- A cadeia reativa faz a busca "funcionar sozinha": digita → `v-model` → `computed` → `v-for` redesenha.
- **`onMounted`** roda ao a tela aparecer — o lugar de carregar os dados da API.

::callout{type="tip" title="Fixando"}
Explique a busca de cupons: *"o `v-model` liga o input ao ref `busca`; o `computed` `filtrados` depende
de `busca` e recalcula ao digitar; o `v-for` renderiza `filtrados`, então a grade se ajusta sozinha"*.
Se você seguiu esse encadeamento reativo, entendeu o coração do Vue.
::
