Um portal tem várias telas, e o usuário anda entre elas. Quem organiza isso é o **roteamento** — e o
Nuxt tem a abordagem mais elegante que existe: **as rotas nascem da estrutura de pastas**, sem você
configurar nada. Nesta aula você entende como isso funciona e como os **layouts** dão a moldura das
telas.

## Roteamento por arquivos: a pasta vira URL

Lembra da convenção da Aula 3? Cada arquivo em **`app/pages/`** vira **automaticamente uma URL**. Olha
a pasta real do portal e as rotas que ela gera:

```text
app/pages/
├── index.vue          →  /               (Visão geral)
├── login.vue          →  /login
├── cupons/
│   ├── index.vue      →  /cupons         (lista)
│   ├── novo.vue       →  /cupons/novo    (criar)
│   └── [id].vue       →  /cupons/:id     (editar — id dinâmico)
├── resgates.vue       →  /resgates
├── faturamento.vue    →  /faturamento
├── relatorios.vue     →  /relatorios
└── empresa.vue        →  /empresa
```

::callout{type="info" title="Sem configurar roteador nenhum"}
No curso do app, o GoRouter exigia declarar cada rota na mão. No Nuxt, você **só cria o arquivo** e a
rota existe. Criou `pages/faturamento.vue`? A URL `/faturamento` já funciona. `index.vue` é o "índice"
da pasta (a raiz `/` ou `/cupons`). Menos código, menos erro — é a convenção trabalhando por você.
::

## Rotas dinâmicas: [id]

E quando a URL tem uma parte variável, como "editar o cupom 42" (`/cupons/42`)? Você nomeia o arquivo
com **colchetes**: `[id].vue`. O `id` vira um parâmetro que a página lê:

```ts
// dentro de app/pages/cupons/[id].vue
const route = useRoute()
const id = Number(route.params.id)   // pega o "42" da URL /cupons/42
```

Assim, **um** arquivo (`[id].vue`) atende `/cupons/1`, `/cupons/2`, `/cupons/qualquer-um` — carregando
o cupom certo pelo id. É o mesmo conceito do `:id` do GoRouter (curso do app) e das rotas REST da API.

## Navegar: NuxtLink e navigateTo

Duas formas de ir de uma tela a outra:

::fields
::field{name="NuxtLink" type="no template (o usuário clica)"}
Um "link" que leva a outra rota, sem recarregar a página (é uma SPA!). Você viu na tela de cupons:
```vue
<NuxtLink to="/cupons/novo" class="dc-btn">Novo cupom</NuxtLink>
```
::
::field{name="navigateTo" type="no script (por código)"}
Navega programaticamente, dentro de uma função. Você viu no login: após entrar, `await navigateTo('/')`
manda para a home.
::
::

::callout{type="tip" title="Por que NuxtLink e não um <a> comum?"}
Um `<a href>` normal faria o navegador **recarregar a página inteira** — matando a fluidez da SPA. O
`NuxtLink` intercepta o clique e troca só o conteúdo, buscando os dados da nova tela — navegação
instantânea, sem piscar. Regra: para navegar dentro do portal, **sempre `NuxtLink`** (ou `navigateTo`
no código); `<a href>` só para links **externos**.
::

## Layouts: a moldura das telas

Repare que quase todas as telas internas do portal compartilham a **mesma moldura**: a sidebar escura à
esquerda, com o conteúdo à direita. Você **não** vai repetir essa sidebar em cada página. Você a coloca
**uma vez** num **layout**, e as páginas entram dentro dele.

Os layouts ficam em `app/layouts/`. O portal tem dois:

::fields
::field{name="default.vue" type="telas internas"}
A moldura padrão: sidebar (navegação, dados da empresa, sair) + área de conteúdo. Todas as telas
logadas usam este.
::
::field{name="auth.vue" type="telas de entrada"}
Uma moldura limpa, sem sidebar, centralizada — para o login. Faz sentido: quem não entrou não vê o menu.
::
::

O layout usa um **`<slot />`** para marcar onde a página entra. Olha o esqueleto do `default.vue` real:

```vue
<template>
  <div class="shell">
    <aside class="sidebar"> ...navegação, empresa, logout... </aside>
    <div class="main">
      <slot />          <!-- ← a PÁGINA atual é renderizada aqui -->
    </div>
  </div>
</template>
```

::callout{type="info" title="slot de layout = o buraco onde a página encaixa"}
É o mesmo conceito de slot da Aula 5, agora no nível da tela inteira: o layout desenha o que é
**comum** (a sidebar) e reserva um `<slot />` para o que **muda** (a página). Trocar de `/cupons` para
`/resgates` só troca o que está **dentro** do slot — a sidebar permanece. Uma moldura, muitas telas.
::

## Escolhendo o layout: definePageMeta

Por padrão, toda página usa o layout `default`. Para uma página usar outro, ela declara com
`definePageMeta`. É assim que o login pede a moldura sem sidebar:

```ts
// no topo de app/pages/login.vue
definePageMeta({ layout: 'auth' })
```

Uma linha, e o login ganha a moldura limpa em vez da sidebar. Simples e explícito.

## Destacar o item ativo

Um detalhe de qualidade no `default.vue`: a sidebar destaca **em qual tela você está**. Ele usa a rota
atual (`useRoute`) para decidir:

```ts
const route = useRoute()
function isActive(to: string) {
  // a home exige match exato; as demais acendem também nas subrotas
  return to === '/' ? route.path === '/' : route.path.startsWith(to)
}
```

Repare no cuidado: `/cupons` fica "ativo" também em `/cupons/novo` (usando `startsWith`), mas a home
`/` exige match **exato** (senão acenderia sempre). Pequenos detalhes assim são o que fazem a navegação
**parecer certa**.

## Resumindo

- No Nuxt, **as rotas nascem da pasta `app/pages/`**: cada arquivo vira uma URL, `index.vue` é a raiz da
  pasta. Sem configurar roteador.
- **Rotas dinâmicas** usam colchetes: `[id].vue` atende `/cupons/:id`, e a página lê `route.params.id`.
- Navegue com **`NuxtLink`** (no template, clicável) ou **`navigateTo`** (no código) — nunca `<a href>`
  interno, que recarregaria a página e mataria a SPA.
- **Layouts** (`app/layouts/`) são as molduras comuns: `default` (com sidebar) e `auth` (login). A
  página entra no **`<slot />`** do layout. `definePageMeta({ layout: 'auth' })` escolhe o layout.
- A sidebar destaca a tela atual comparando `route.path` (com cuidado: exato para a home, `startsWith`
  para as demais).

::callout{type="tip" title="Fixando"}
Explique como `/cupons/novo` funciona: *"o arquivo `pages/cupons/novo.vue` gera a rota; um `NuxtLink`
leva até ela sem recarregar; ela usa o layout `default` (a sidebar), aparecendo dentro do `<slot />`; e
a sidebar acende 'Cupons' porque a rota começa com `/cupons`"*. Se você amarrou pasta → rota → layout,
domina a estrutura.
::
