Um site de marketing tem uma missão: **converter**. Cada página é uma **landing** desenhada para levar
o visitante a uma ação (baixar o app, virar parceiro). Nesta última página do guia, você vê a anatomia
de uma landing e um recurso avançado do Vue que dá vida a ela: uma **diretiva customizada** de
animação.

## A anatomia de uma landing

Uma página de conversão segue uma estrutura mais ou menos previsível — uma sequência de **seções** que
conduzem o visitante do "o que é isso?" até o "quero!":

::steps
### Hero (o topo)
A primeira dobra: um título forte, um subtítulo e o **CTA** principal. É o que aparece antes de rolar —
tem que "fisgar" em segundos.

### Prova de valor
Benefícios, números ("+10 mil assinantes"), como funciona. Responde "por que eu deveria me importar?".

### Prova social / confiança
Depoimentos, parceiros, selos. Responde "isso é confiável?".

### CTA final
Um último convite claro para agir, geralmente repetindo o do topo.
::

::callout{type="info" title="CTA: o botão que é o ponto de tudo"}
**CTA** (Call To Action, "chamada para ação") é o botão/link que representa o objetivo da página:
"Baixar o app", "Quero ser parceiro". Tudo na landing existe para levar o olho até o CTA. Num painel, o
usuário já entrou e tem tarefas; numa landing, ele é um visitante que você precisa **convencer** — por
isso o CTA é destacado, repetido e sempre visível. É a diferença de mentalidade entre "ferramenta" e
"vitrine de conversão".
::

Cada seção é montada com os componentes e o design system que você já conhece (`SiteHeader`,
`SiteFooter`, cards, variáveis CSS). A novidade está em como elas **ganham vida** ao rolar.

## Diretivas customizadas: criando o seu v-

No curso web (Aula 6) você usou diretivas prontas: `v-if`, `v-for`, `v-model`. O Vue também deixa você
**criar as suas** — e o site tem uma ótima: **`v-reveal`**, que faz um elemento **surgir com animação**
quando ele entra na tela ao rolar.

```vue
<div v-reveal>Este bloco aparece suavemente ao rolar até ele</div>
<div v-reveal="120">Este aparece com 120ms de atraso (efeito cascata)</div>
```

A diretiva é registrada num **plugin** (`app/plugins/reveal.ts`) — código que roda na inicialização do
app (lembra da pasta `plugins/` da Aula 4 do curso web). Ele usa o **IntersectionObserver** do
navegador, que avisa quando um elemento entra na área visível:

```ts
// app/plugins/reveal.ts (essência)
const observer = new IntersectionObserver((entries, obs) => {
  for (const entry of entries) {
    if (entry.isIntersecting) {              // o elemento entrou na tela?
      entry.target.classList.add('is-visible') // ativa a animação (via CSS)
      obs.unobserve(entry.target)             // anima uma vez só
    }
  }
}, { threshold: 0.12 })

nuxtApp.vueApp.directive('reveal', {
  mounted(el, binding) {
    // ...só esconde e observa elementos que estão ABAIXO da dobra...
    el.classList.add('reveal')
    observer.observe(el)
  },
})
```

::callout{type="tip" title="Por que uma diretiva, e não um componente?"}
Animar a entrada é um **comportamento** que você quer aplicar a **qualquer** elemento (um título, um
card, uma seção) sem envolvê-lo num componente extra. Diretiva é a ferramenta certa para isso: você só
adiciona `v-reveal` no elemento e pronto. Regra: **componente** para uma peça de UI reutilizável;
**diretiva** para um comportamento que você "pincela" em elementos existentes. Saber criar uma diretiva
é um passo de front-end mais avançado — e este site mostra um uso real e elegante.
::

## O cuidado com a hidratação (de novo)

Lembra do "hydration mismatch" da página anterior? O `reveal.ts` foi escrito **exatamente** para
evitá-lo, e os comentários no código explicam:

::fields
::field{name="No servidor: não esconde nada"}
O `getSSRProps()` retorna `{}` — ou seja, o HTML do servidor sai **totalmente visível**, sem a classe de
"escondido". Assim o conteúdo aparece na hora (bom para o LCP) e o servidor e o cliente **batem** (sem
mismatch).
::
::field{name="No cliente: só esconde o que está fora da tela"}
Ao montar, a diretiva **só** esconde-e-observa elementos que estão **abaixo da dobra**. O que já está
visível no topo **permanece visível** — nada "pisca" se escondendo depois de aparecer.
::
::

::callout{type="info" title="Isso é código maduro"}
Uma animação de entrada malfeita causa dois bugs clássicos: o conteúdo **some** por um instante depois
de já ter aparecido (flash feio), ou o Vue reclama de mismatch e redesenha. O `reveal.ts` resolve os
dois com cuidado — servidor sempre visível, cliente só animando o que está fora da tela. É o tipo de
detalhe que separa "animação que impressiona no vídeo" de "animação que funciona para milhares de
visitantes reais, com SEO e performance intactos". Atenção a esses detalhes **é** senioridade.
::

## Resumindo

- Um site de marketing é feito de **landings**: seções (hero, prova de valor, prova social, CTA final)
  que conduzem o visitante à ação.
- O **CTA** ("Baixar o app", "Ser parceiro") é o ponto de tudo — destacado e repetido, porque o
  visitante precisa ser **convencido** (diferente de um painel, onde ele já entrou).
- O Vue permite **diretivas customizadas**: o site tem `v-reveal`, que anima a entrada dos elementos na
  rolagem usando **IntersectionObserver**, registrada num **plugin**. Diretiva = comportamento
  "pincelado" em qualquer elemento (vs componente = peça de UI).
- O `reveal.ts` cuida da **hidratação**: no servidor não esconde nada (visível + sem mismatch), no
  cliente só anima o que está **fora da tela** (sem flash). Detalhe de código maduro.
