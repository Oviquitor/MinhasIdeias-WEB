| GET | /indicacoes/me | Resumo indicações |
| POST | /notificacoes/tokens | Registra FCM |
| DELETE | /notificacoes/tokens | Remove FCM |
| GET | /health | Health check |


## 20.1 Padrão de resposta

```json
{
  "success": true,
  "message": "Operação realizada com sucesso",
  "data": {},
  "errors": []
}
```

## 20.2 Padrão de erro

```json
{
  "success": false,
  "message": "Dados inválidos",
  "errors": [
    {"field": "email", "message": "E-mail obrigatório"}
  ]
}
```

## 20.3 Regras da API

- Validar tudo no backend.
- Não confiar em valores do app.
- Usar DTOs.
- Padronizar erros.
- Versionar futuramente com `/v1`.
- Proteger endpoints por roles.
- Usar paginação.
- Logar eventos críticos.

# 23. LGPD, privacidade e segurança

## 23.1 Dados pessoais possíveis

- Nome.
- E-mail.
- Telefone.
- Cidade.
- Histórico de cupons.
- Favoritos.
- Tokens de notificação.
- Dados de pagamento processados pelo provedor.
- Documento, se exigido pelo pagamento.
- Dados de suporte.
- Localização opcional, se implementada.

## 23.2 Princípios

- Coletar o mínimo necessário.
- Explicar finalidade.
- Proteger dados.
- Permitir exclusão.
- Permitir correção.
- Ter política de privacidade.
- Ter termos de uso.
- Ter contrato com parceiros.
- Controlar acesso.

## 23.3 Permissões mobile

- Câmera: QR code.
- Notificações: alertas.
- Localização: ofertas próximas, se for usada.

Regras:

- Pedir permissão no momento certo.
- Explicar motivo.
- Não bloquear uso básico se localização for negada.
- Permitir cidade manual.

## 23.4 Segurança técnica

- HTTPS obrigatório.
- JWT seguro.
- Hash forte de senha.
- Token em FlutterSecureStorage.
- Dados não sensíveis em SharedPreferences.
- Rate limit.
- Logs sem dados sensíveis.
- CORS restrito.
- Secrets em variáveis de ambiente.
- Backups.
- Auditoria.
- Monitoramento.

# 26. MVP

- Parceiros suficientes na primeira cidade.

- Resgate funcionando.
- Usuários entendendo valor.
- Parceiros cumprindo ofertas.
- Baixo índice de erro.
- Histórico de economia gerando percepção de retorno.

# 30. Copy e textos prontos

## 30.1 Pitch para usuário

> “Com o Descontai você encontra descontos reais em lugares da sua cidade, usa cupons pelo app e acompanha quanto economizou.”

## 30.2 Pitch para parceiro

> “O Descontai é um clube local de benefícios. A pessoa assina o app para acessar descontos e experiências na cidade. Sua empresa entra com ofertas controladas, com regras e limites, e acompanha os resgates. Não substitui Instagram nem WhatsApp; é mais um canal para as pessoas descobrirem seu negócio.”

## 30.3 Empty states

Sem ofertas:

> “Ainda não temos ofertas nessa categoria. Estamos trazendo novos parceiros para sua cidade.”

Sem histórico:

> “Quando você usar seu primeiro cupom, sua economia aparecerá aqui.”

Sem assinatura:

> “Assine para usar os cupons premium e começar a economizar.”

## 30.4 Erros úteis

- “Esse cupom já foi usado.”
- “Oferta fora do horário permitido.”
- “Sua assinatura não está ativa.”
- “Pagamento ainda não confirmado.”
- “Não encontramos esse código.”
- “Você não pode usar seu próprio código de indicação.”


# 31. Riscos e mitigação

## 31.1 Poucos parceiros

Mitigação:

- Fechar parceiros antes do lançamento.
- Começar em uma cidade.
- Priorizar ofertas fortes.
- Fazer curadoria manual.

## 31.2 Oferta sem valor

Mitigação:

- Revisar desconto real.
- Destacar economia estimada.
- Medir resgates.
- Remover ofertas ruins.

## 31.3 Parceiro não cumpre oferta

Mitigação:

- Contrato.
- Regras claras.
- Suporte.
- Avaliação.
- Pausar parceiro reincidente.

## 31.4 Fraude

Mitigação:

- QR dinâmico.
- Código temporário.
- Limites.
- Auditoria.
- Antifraude.

## 31.5 Pagamento falho

Mitigação:

- Webhook robusto.
- Reprocessamento Admin.
- Logs.
- Consulta no provedor.
- Mensagens claras.

## 31.6 Expansão rápida demais

Mitigação:

- Playbook por cidade.
- Métricas mínimas.
- Parceiros ativos antes de abrir cidade.
