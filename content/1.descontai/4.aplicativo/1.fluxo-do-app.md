## 1. Visão Geral

O **Descontai** é um clube de descontos. Usuários assinam um plano e ganham acesso a cupons de estabelecimentos parceiros (gastronomia, serviços, etc.). Os cupons são resgatados presencialmente via código ou QR code. O app também possui sistema de indicação com saldo monetário para indicadores.

**Papéis do usuário:**
- **Usuário padrão:** Acessa cupons, vê histórico, gerencia assinatura.
- **Indicador:** Idem + tem código de indicação e saldo de indicações visível no perfil.

---

## 2. Arquitetura de Sessão e Segurança

A sessão é gerenciada pelo `SessionCache` — uma camada estática com três níveis de armazenamento:

| Dado | Armazenamento | Motivo |
|---|---|---|
| Token JWT | `FlutterSecureStorage` (criptografado) | Segurança máxima |
| Token biométrico | `FlutterSecureStorage` | Separado do token principal |
| userId, cidadeId, cidade, nome, email | `SharedPreferences` | Acesso rápido, não sensível |
| codigoIndicacao, saldoIndicacao | `SharedPreferences` | Cache local |
| Preferência de "lembrar", "biometria" | `SharedPreferences` | Persistência de UX |

**Fluxo de autenticidade no GoRouter:**
- `SessionCache.listenable` é um `ValueNotifier` que notifica o router a cada mudança de sessão.
- Qualquer tela que não seja de autenticação redireciona para `/login` se não houver token.
- Se o usuário estiver logado e tentar acessar `/login`, é redirecionado para `/base`.

## 3. Fluxo de Inicialização (Splash)

**Tela:** `SplashPage` — Rota `/splash`

O usuário vê o logo centralizado em fundo branco por um instante. Não há interação manual — a tela decide sozinha para onde ir.

```
App abre
│
├── SessionCache.restore()  ← lê token, prefs e estado da sessão
│
├── [Tem token + remember = true]
│     ├── HomeController.carregarDados()
│     ├── UserController.sincronizarValidadeUsuario()
│     └── → /base
│
├── [Tem token + biometria habilitada]
│     └── → /login  (LoginPage vai pedir biometria automaticamente)
│
└── [Sem token]
      └── → /welcome
```

**API chamada:** nenhuma diretamente (dados carregados pelos controllers).

### 4.2 Login

**Rota:** `/login`

**O que o usuário vê:**
- Logo do app no topo (40% da altura).
- Campo e-mail (texto em minúsculas, sem espaços automaticamente).
- Campo senha com botão de mostrar/esconder.
- Checkbox "Lembrar de mim".
- Botão "Entrar".
- Link "Esqueci minha senha" → `/redefinir-senha`.

**Fluxo ao clicar em "Entrar":**

```
1. Valida campos (e-mail e senha não vazios)
2. AuthService.login(email, senha, lembrar)
   └── POST /auth/login
3. Sucesso:
   ├── SessionCache.saveAuth(token, userId, ...)
   ├── Se remember = true → salva token criptografado + bloqueia biometria
   ├── HomeController.carregarDados()
   ├── UserController.sincronizarValidadeUsuario()
   ├── NotificationService.registerToken(fcmToken)  ← registra push
   └── → /base
4. Erro:
   └── Exibe TopNotice com mensagem (ex: "E-mail ou senha inválidos")
```

**Biometria automática (se habilitada):**

```
LoginPage abre
│
├── [biometricEnabled = true + hasBiometricToken = true]
│     └── BiometricService.authenticate()
│           ├── Sucesso → SessionCache.unlockBiometricSession()
│           │             HomeController.carregarDados()
│           │             → /base
│           └── Falha (até 5x) → libera formulário manual
│
└── [Sem biometria] → formulário normal
```

### 4.3 Cadastro

**Rota:** `/register`

**O que o usuário vê:**
- Título: "Que bom te ver por aqui! Crie sua conta..."
- Campo Nome.
- Campo Cidade (autocomplete — lista de cidades via `GET /cidades`).
- Campo E-mail.
- Campo Senha com toggle de visibilidade.
- Botão "Cadastrar".

**Fluxo ao clicar em "Cadastrar":**

```
1. Valida campos (todos obrigatórios; cidade deve ter sido selecionada no dropdown)
2. AuthService.register(email, senha, nome, cidadeId, cidade)
   └── POST /auth/register
3. Sucesso:
   ├── Auto-login: AuthService.login(email, senha, lembrar=false)
   │   └── POST /auth/login
   ├── HomeController.carregarDados()
   ├── UserController.sincronizarValidadeUsuario()
   └── → /base
4. Erro (ex: e-mail já cadastrado):
   └── Exibe TopNotice com mensagem de erro
```

### 4.4 Redefinir Senha

**Rota:** `/redefinir-senha`

Fluxo em **3 etapas** com indicador de progresso (● — ● — ●).

**Etapa 1 — Solicitar código:**
- Campo e-mail.
- Botão "Enviar código" → `POST /auth/esqueci-senha`
- Sucesso: avança para etapa 2, mostra e-mail mascarado (ex: `m**@example.com`).

**Etapa 2 — Redefinir:**
- Mostra e-mail mascarado + link "Trocar" (volta à etapa 1).
- Campo "Código" (mínimo 4 caracteres).
- Campo "Nova senha" (mínimo 6 caracteres) com toggle.
- Campo "Confirmar senha" (deve ser igual).
- Botão "Redefinir senha" → `POST /auth/redefinir-senha` com `{token, novaSenha}`.
- Sucesso: avança para etapa 3.

**Etapa 3 — Confirmação:**
- Banner verde: "Senha redefinida com sucesso!"
- Botão "Voltar para o login" → `/login`.

## 7. Busca e Filtros

**Tela:** `SearchPage` — rota `/search/:tipo`

O parâmetro `:tipo` pré-seleciona um filtro ao abrir (ex: "Gastronomia"). Se vazio ou "Todos", nenhum filtro é pré-aplicado.

### O que o usuário vê

**Barra de busca (AppBar):**
- Ícone de busca.
- Campo de texto "Buscar por cupons".
- Botão X de limpar (só aparece quando há texto digitado).
- Botão "voltar" (só aparece se veio de uma categoria específica).

**Chips de filtro (scroll horizontal):**
- Fixos: "Todos", "Gostei", "Disponíveis".
- Dinâmicos: todos os tipos de produto e categorias existentes nos produtos carregados.
- Chip selecionado fica com fundo preenchido.
- **Tap em chip** → seleciona filtro + dispara busca imediata.

**Lista de resultados (scroll vertical):**
- Card de produto com imagem, nome, desconto, empresa.
- Botão coração para curtir/descurtir.
- **Tap no card** → detalhe do produto.

### Lógica de busca

A busca é realizada pelo `PesquisaController.buscar()` com **debounce de 220ms** na digitação.

```
PesquisaController.buscar(todos, likedIds, termo, filtro)
│
├── Filtro por tipo:
│   ├── "gostei"      → apenas produtos em likedIds
│   ├── "disponiveis" → apenas produtos com disponivelParaUso = true
│   ├── "gastronom*"  → categoria contém "gastronom"
│   ├── "servico*"    → categoria contém "servico"
│   └── outro         → match por tipoProduto, depois por categoria
│
└── Filtro por termo (após filtro de tipo):
    └── normalize(produto.nome).contains(normalize(termo))
        └── normalize = lowercase + remover acentos
```

**Resultado exibido:**
- Se há texto digitado OU filtro ativo (diferente de "Todos") → `search.resultados`
- Se campo vazio E filtro "Todos" → `home.produtos` (lista completa)

## 8. Detalhe do Produto e Uso de Cupom

**Rota:** `/produto/:id`  
Recebe o `id` na URL e, opcionalmente, o objeto `ProdutosResponse` como `extra` (evita chamada extra à API).

### O que o usuário vê

**Hero do produto (topo):**
- Imagem full-width do produto.
- Botão coração no canto superior direito.
- Logo da empresa sobreposta.
- Badge com valor do desconto.

**Painel de detalhes (80% inferior, sobre o hero):**
- Nome do produto.
- Nome da empresa.
- Valor do desconto em destaque.
- **"Precisa saber":** Instruções de uso resumidas.
- **"Como usar":** Passo a passo.
- **"Regras":** Termos e condições.
- **"Localização":** Endereço do estabelecimento.
- **Horários:** Parsed de string JSON pelo `HorarioUtils` — exibe "Aberto agora" ou horários do dia.

**Botão de ação fixo (rodapé):**

| Condição | Texto do botão | Ação |
|---|---|---|
| Sem assinatura ativa | "Ativar assinatura" | → `/assinatura` |
| Assinatura ativa, cupom não usado | "Usar cupom" | Abre bottom sheet |
| Cupom já usado | "Cupom usado" | Desabilitado |

### Fluxo "Usar Cupom" (bottom sheet)

```
Usuário toca "Usar cupom"
│
└── Bottom sheet abre com:
    ├── Campo "Código do cupom" (pré-preenchido com código do produto)
    ├── [Se valorDesconto <= 0] Campo "Valor do desconto" (usuário informa)
    └── Botão QR → abre modal de scanner
         └── MobileScanner full-screen
             ├── Aponta câmera para QR code do estabelecimento
             ├── Detecta → preenche campo "Código" automaticamente
             └── Fecha modal

Usuário toca "Aplicar cupom"
├── Valida campos
├── HomeService.usarCupom(codigoCupom, valorDesconto)
│   └── POST /cupons/usar
├── Sucesso:
│   ├── HomeController.recarregar()  ← atualiza lista de disponíveis
│   ├── Exibe TopNotice verde "Cupom aplicado!"
│   └── Fecha bottom sheet
└── Erro:
    └── Exibe TopNotice vermelho com mensagem
```

**Botão coração:**
- `UserController.toggleLike(produtoId)`
- Estado persistido em SharedPreferences.
- Refletido em tempo real em todas as telas (Home, Busca, Detalhe).

## 9. Histórico de Cupons

**Tela:** `HistoricoPage` — aba 3

Exibe todos os cupons que o usuário já usou, agrupados por mês em ordem decrescente. Os dados vêm do `HomeController.historico` (carregado junto com os dados da home — sem chamada extra).

### O que o usuário vê

**Card de economia total (topo):**
- Valor total economizado: ex. "R$ 245,00".
- Grid 2 colunas com os últimos 2 meses + valores.

**Timeline por mês:**
Para cada mês (ex: "jan. 2024"):
- Cabeçalho com nome do mês.
- Lista de itens com linha vertical conectora (timeline visual).
- Cada item: imagem do produto, nome, valor do desconto, data/hora de uso formatada (`dd/MM/yyyy às HH:mm`).
- Rodapé do mês: "Total no período: R$ XX,00".

**Cálculos:**
- Total geral: `historico.fold((acc, p) => acc + p.produto.valorDesconto)`
- Agrupamento: por label de mês abreviado em pt-BR (ex: "jan. 2024")

### 12.4 Pagamento

**Rota:** `/assinatura/pagamento`

Recebe o plano selecionado via `extra`.

**O que o usuário vê:**

**Resumo do plano** (card no topo): ícone, nome, descrição, valor.

**Seletor de método de pagamento** (3 opções):

| Opção | Ícone | Método |
|---|---|---|
| Pix | QR code | Gera QR + código copia-e-cola |
| Crédito | Cartão | Formulário de cartão + parcelamento |
| Débito | Cartão | Formulário de cartão sem parcelamento |

**Formulário de cartão (crédito/débito):**
- Nome do titular
- Número do cartão (apenas dígitos, máx 19 caracteres)
- CVV (3-4 dígitos, oculto)
- Validade (MM/AA)
- Parcelas (1–12x, apenas para crédito)

**Dados do comprador (cartão ou Pix):**
- E-mail (obrigatório)
- Tipo de documento: CPF / CNPJ
- Número do documento (obrigatório)
- Telefone (obrigatório)
- CEP / Código postal (8 dígitos, obrigatório) — apenas para cartão
- Número do endereço (obrigatório) — apenas para cartão
- Complemento (opcional) — apenas para cartão

**Código de indicação (opcional):**
- Campo visível para todos.
- Validação: usuário não pode usar o próprio código.
- Se preenchido, é enviado ao backend.

**Botão de pagamento:**

```
[Pix] → "Gerar Pix"
  └── POST /pagamentos/asaas/pix
      Resposta:
      ├── status (ex: "PENDING")
      ├── pixQrCode (base64 para exibir QR)
      └── pixCopiaECola (string copiável)
      ──→ Exibe QR + botão "Copiar código"

[Cartão] → "Pagar [Nome do Plano]"
  └── POST /pagamentos/asaas/cartao
      ──→ Sucesso:
          ├── UserController.sincronizarValidadeUsuario()
          └── → /assinatura
```

**Validações do formulário:**
- E-mail: contém "@"
- Número do cartão: mínimo 13 dígitos
- CVV: mínimo 3 dígitos
- Validade: formato MM/AA válido
- Documento: mínimo 5 caracteres
- Telefone: mínimo 8 dígitos

## 14. Biometria

### Habilitar biometria

1. Usuário vai em **Dados da Conta** → ativa toggle "Login com biometria".
2. App verifica: `BiometricService.isSupported()` + `canCheckBiometrics()`.
3. Se suportado:
   - Salva preferência (`biometricEnabled = true`).
   - Move token principal para `_lockedBiometricToken` (secure storage, chave diferente).
   - Limpa token principal da memória.
4. Se não suportado: toggle volta para falso, exibe alerta.

### Login com biometria

```
LoginPage.initState()
├── [biometricEnabled = true + _lockedBiometricToken existe]
│     └── BiometricService.authenticate(reason: "Use sua biometria para entrar")
│           ├── Sucesso:
│           │     ├── SessionCache.unlockBiometricSession()
│           │     │   └── token = _lockedBiometricToken
│           │     ├── HomeController.carregarDados()
│           │     └── → /base
│           └── Falha (até 5 tentativas):
│                 └── Libera formulário de login manual
└── [Sem biometria] → formulário exibido diretamente
```

### Desabilitar biometria

- Toggle → `SessionCache.updateBiometric(enabled: false)`.
- Token biométrico bloqueado é movido de volta para token principal.

## 15. Push Notifications (FCM)

### Fluxo de registro

```
main() → Firebase.initializeApp()
       → FirebaseMessaging.requestPermission()
       → messaging.getToken()
           └── [Token obtido + SessionCache.token não nulo]
                 └── NotificationService.registerToken(fcmToken)
                     └── POST /notificacoes/tokens
```

### Refresh de token

```
messaging.onTokenRefresh.listen((newToken)
  └── [SessionCache.token não nulo]
        └── NotificationService.registerToken(newToken)
```

### Listeners de mensagem

| Evento | Situação | Ação atual |
|---|---|---|
| `onMessage` | App aberto, notificação recebida | Log em debug mode |
| `onMessageOpenedApp` | App em background, usuário toca notificação | Log em debug mode |
| `onBackgroundMessage` | App fechado, notificação recebida | Inicializa Firebase |

### Logout

- `NotificationService.removeToken()` → `DELETE /notificacoes/tokens` — desregistra o dispositivo antes de limpar a sessão.

## 16. Tratamento de Erros

### Erros de rede (Dio)

| Tipo de erro | Mensagem exibida |
|---|---|
| `connectionTimeout` / `sendTimeout` / `receiveTimeout` | "Tempo esgotado. Verifique sua conexão." |
| `connectionError` / `SocketException` | "Sem conexão com a internet." |
| `badCertificate` | "Conexão insegura detectada." |
| HTTP 400 | "Dados inválidos." |
| HTTP 401 / 403 | "E-mail ou senha inválidos." (auth) / "Acesso negado." (outros) |
| HTTP 409 | "E-mail já cadastrado." |
| HTTP 422 | "Dados inválidos. Verifique o formulário." |
| HTTP 500+ | "Erro interno do servidor. Tente novamente." |

### Erros de Firebase

- `FlutterError.onError` → `FirebaseCrashlytics.recordFlutterFatalError()`.
- `PlatformDispatcher.onError` → `FirebaseCrashlytics.recordError(..., fatal: true)`.

### Exibição na UI

**`TopNotice` widget** — banner flutuante que desce do topo da tela:
- 🔴 **Vermelho** — erros
- 🟢 **Verde** — sucessos
- 🔵 **Azul** — informações
- Auto-fecha em ~3-4 segundos ou ao tocar.

| Propriedade | Tipo | Descrição |
|---|---|---|

| `gastronomia` | `List<ProdutosResponse>` | Filtro por categoria gastronômica |
| `servicos` | `List<ProdutosResponse>` | Filtro por categoria de serviços |
| `categorias` | `List<CategoriaResponse>` | Lista de categorias |
| `tiposProduto` | `List<TipoProdutoResponse>` | Lista de tipos de produto |

| `errorMessage` | `String?` | Último erro |
| `cuponsDisponiveis` | `int` (getter) | Quantidade de produtos disponíveis |
| `cuponsUsados` | `int` (getter) | Tamanho do histórico |

### `UserController`
Gerencia o estado do usuário: assinatura e favoritos.

| Propriedade | Tipo | Descrição |
|---|---|---|
| `assinaturaValidadeIso` | `String?` | ISO 8601 da validade da assinatura |
| `assinaturaAtiva` | `bool` (getter) | `true` se validade > hoje (até 23:59:59) |
| `assinaturaValidadeData` | `DateTime?` (getter) | Data parseada da validade |
| `likedIds` | `Set<int>` | IDs de produtos curtidos |
| `favoritosCount` | `int` (getter) | Quantidade de favoritos |

Métodos: `toggleLike(id)`, `isLiked(id)`, `sincronizarUsuario()`, `sincronizarValidadeUsuario()`, `atualizarValidade(iso)`.

### `PesquisaController`
Gerencia o estado de busca.

| Propriedade | Tipo | Descrição |
|---|---|---|
| `resultados` | `List<ProdutosResponse>` | Produtos filtrados |
| `termoBusca` | `String` | Termo atual digitado |
| `filtroAtivo` | `String?` | Filtro de categoria ativo |

Métodos: `buscar(todos, likedIds, termo, filtro)`, `limpar()`.
