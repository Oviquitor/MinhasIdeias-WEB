Na Aula 3 você viu que a interface é feita de **componentes** (peças reutilizáveis). Agora vamos
**criar** um: como receber dados de fora (props), definir valores padrão e reaproveitar a mesma peça em
vários lugares. Vamos usar o `DcKpi` real — o card de indicador dos dashboards.

## Por que componentes

Imagine a Visão Geral do portal: ela mostra quatro cards de indicador ("Resgates no mês", "Cupons
ativos"…) — todos com o mesmo formato, só mudando o texto e o número. Você **não** vai copiar e colar o
HTML/CSS quatro vezes. Você cria **um** componente `DcKpi` e o usa quatro vezes, passando dados
diferentes.

::callout{type="info" title="Componente = fôrma de biscoito"}
Um componente é uma **fôrma de biscoito**: você faz a fôrma **uma vez** (a estrutura + o estilo do
card) e usa ela para assar vários biscoitos, cada um com um recheio diferente (os dados). Muda o
recheio, não a fôrma. Se você quiser mudar o formato do card, muda **a fôrma** — e todos os biscoitos
mudam juntos. Reuso e consistência de graça.
::

## Passando dados: props

**Props** ("properties") são os dados que um componente **recebe de fora**, de quem o usa. É como
"passar argumentos para uma função", só que para uma peça de interface. Olha como a Visão Geral usa o
`DcKpi`:

```vue
<DcKpi label="Resgates no mês" value="1.284" delta="↑ 12%" icon="users" tone="green" />
<DcKpi label="Cupons ativos"   value="8"     icon="ticket" tone="brand" :loading="carregando" />
```

Cada atributo (`label`, `value`, `icon`, `tone`) é uma **prop**. O mesmo componente, dados diferentes,
resultados diferentes.

::callout{type="tip" title="O detalhe dos dois-pontos (:)"}
Repare: `label="Resgates no mês"` (sem `:`) passa o **texto literal**. Já `:loading="carregando"` (com
`:`) passa o **valor de uma variável** — o `:` é a forma curta de `v-bind`, que diz "isto é uma
expressão JavaScript, não texto". Sem o `:`, `loading="carregando"` passaria a **palavra**
"carregando" (uma string). Com o `:`, passa o **valor** da variável `carregando` (true/false). Essa
distinção confunde todo iniciante — grave: **`:` significa "avalie como código"**.
::

## Definindo as props: defineProps

Dentro do componente, você declara **quais** props ele aceita e de que tipo, com `defineProps`. Olha o
`DcKpi.vue` real:

```vue
<script setup lang="ts">
withDefaults(
  defineProps<{
    label: string          // rótulo (ex.: "Resgates no mês")
    value: string          // valor já formatado (ex.: "1.284")
    delta?: string         // variação opcional (o ? = pode não vir)
    icon: string           // nome do ícone
    tone?: 'brand' | 'green' | 'amber' | 'blue'  // cor, só estes valores
    loading?: boolean      // se true, mostra skeleton
  }>(),
  { tone: 'brand' },       // valor PADRÃO: se não passarem 'tone', usa 'brand'
)
</script>
```

Duas coisas importantes:

::fields
::field{name="Tipos nas props"}
Cada prop tem um tipo (`string`, `boolean`). O `tone` só aceita quatro valores exatos
(`'brand' | 'green' | ...`) — se você passar `tone="roxo"`, o TypeScript **avisa o erro na hora**. É o
TS da Aula 2 protegendo o componente.
::
::field{name="Opcional (?) e padrão (withDefaults)"}
O `?` marca a prop como **opcional** (`delta?`, `loading?`). E o `withDefaults(..., { tone: 'brand' })`
define um **valor padrão**: quem não passar `tone` recebe `'brand'`. Assim o componente funciona mesmo
com o mínimo de informação.
::
::

## Usando as props no template

Declaradas as props, o template as usa — com interpolação (`{{ }}`) e diretivas. Recorte do `DcKpi`:

```vue
<template>
  <div class="dc-card kpi">
    <div class="top">
      <span class="dc-label">{{ label }}</span>   <!-- mostra a prop 'label' -->
      <span class="icon" :style="{ background: toneMap[tone].bg, color: toneMap[tone].fg }">
        <DcIcon :name="icon" :size="18" />        <!-- passa 'icon' para OUTRO componente -->
      </span>
    </div>
    <!-- Enquanto carrega, um retângulo shimmer; senão, o valor. -->
    <div v-if="loading" class="dc-skeleton" />
    <div v-else class="value">{{ value }}</div>
    <!-- A variação só aparece se veio e não está carregando. -->
    <div v-if="delta && !loading" class="delta">{{ delta }}</div>
  </div>
</template>
```

Repare que o `DcKpi` **usa outro componente dentro dele** (`<DcIcon>`), passando a prop `icon` adiante.
Componentes dentro de componentes — a árvore da Aula 2, agora com peças reutilizáveis. (O `v-if`/`v-else`
você vê a fundo na próxima aula.)

::callout{type="info" title="Auto-import: nem precisou importar"}
Note que o template usa `<DcIcon>` sem nenhum `import` no `<script>`. Lembra da convenção do Nuxt
(Aula 3)? Todo componente em `app/components/` é **auto-importado**. Você cria o `DcKpi.vue` e já pode
escrever `<DcKpi />` em qualquer template — o Nuxt acha e conecta. Menos cerimônia, mais foco na tela.
::

## Slots: quando o "recheio" é conteúdo

Props passam **dados** (texto, número). Mas às vezes você quer passar **conteúdo** (outros elementos)
para dentro de um componente — por exemplo, os botões do rodapé de um modal. Para isso existem os
**slots**: "buracos" no componente onde quem o usa **encaixa** conteúdo. Você vê isso no `DcModal` e no
`DcTopbar`:

```vue
<DcTopbar title="Novo cupom">
  <template #actions>                 <!-- encaixa no slot chamado "actions" -->
    <NuxtLink to="/cupons" class="dc-btn">Voltar</NuxtLink>
  </template>
</DcTopbar>
```

O `DcTopbar` desenha o título e reserva um slot `actions`; a página **preenche** esse slot com os
botões que quiser. É reuso levado adiante: o mesmo topo, com ações diferentes por tela.

::callout{type="tip" title="Props x slots, sem decoreba"}
**Props** = você passa **valores** (o `label`, o `value`). **Slots** = você passa **conteúdo/elementos**
(botões, textos formatados). Regra prática: se é um dado simples, prop; se é "um pedaço de tela que
varia", slot. O `DcKpi` usa props; o `DcModal` e o `DcTopbar` usam slots para o conteúdo variável.
::

## Resumindo

- **Componentes** são peças reutilizáveis (a "fôrma de biscoito"): faça uma vez, use muitas, com dados
  diferentes.
- **Props** são os dados que o componente recebe de fora. No uso, `label="x"` passa texto literal e
  **`:prop="var"`** (v-bind) passa o **valor** de uma variável.
- Dentro do componente, **`defineProps`** declara as props com **tipos**; **`withDefaults`** dá valores
  **padrão**; o **`?`** marca opcionais. O TypeScript avisa erros na hora.
- Componentes usam outros componentes no template (e o Nuxt **auto-importa** os de `app/components/`).
- **Slots** passam **conteúdo** (não só dados) para dentro do componente — o `#actions` do `DcTopbar`,
  o rodapé do `DcModal`.

::callout{type="tip" title="Fixando"}
Pense na Visão Geral: *"são quatro `DcKpi`, a mesma fôrma, cada um recebendo `label`, `value` e `tone`
diferentes por props"*. E note: *"o `:loading` tem dois-pontos porque passa o valor de uma variável, não
a palavra"*. Se você entendeu componente = fôrma e props = recheio, domina o reuso de interface.
::
