## 8. Jornada completa do lojista

### 8.1 Antes do Vitrine

O lojista normalmente divulga seus produtos assim:

```text
Foto no Instagram
Status no WhatsApp
Mensagem manual
PDF desatualizado
Álbum solto no celular
```

O problema é que nada disso vira um catálogo realmente navegável.

### 8.2 Primeira experiência no Vitrine

A jornada ideal deve ser:

1. Lojista acessa a landing page.
2. Entende em poucos segundos o que é o Vitrine.
3. Clica em criar minha vitrine.
4. Cria conta com nome, e-mail, telefone e senha.
5. Informa nome da loja.
6. Escolhe o tipo de negócio.
7. Escolhe um modelo visual.
8. Adiciona logo ou pula essa etapa.
9. Define cor principal ou aceita sugestão.
10. Cadastra o primeiro produto ou serviço.
11. Visualiza a vitrine publicada.
12. Copia o link para divulgar.
13. Recebe primeiras intenções via WhatsApp.

### 8.3 Onboarding ideal

O onboarding deve ser guiado e dividido em etapas curtas:

```text
Etapa 1 — Sua loja
Etapa 2 — Aparência
Etapa 3 — Primeiro produto
Etapa 4 — WhatsApp
Etapa 5 — Publicar
```

O sistema deve mostrar progresso:

```text
Sua vitrine está 60% pronta
Falta adicionar WhatsApp e publicar primeiro produto
```

### 8.4 Ativação do lojista

O lojista só percebe valor quando consegue compartilhar uma página real.

Portanto, a métrica de ativação não é apenas cadastro. É:

```text
Lojista criou loja + cadastrou produto + publicou link
```

### 8.5 Retenção

O lojista continuará pagando se o Vitrine virar parte da rotina dele:

- Atualizar produtos.
- Criar promoções.
- Enviar link para clientes.
- Consultar pedidos.
- Ver produtos mais acessados.
- Controlar custo e margem.
- Usar IA para gerar descrições e campanhas.


ou por domínio próprio:

- Filtrar por categoria.
- Ver promoções.
- Abrir produto.
- Ver fotos.
- Ver preço.
- Ver variações.

```text
Olá! Vim pelo catálogo da Loja Bella.

1. Camiseta Oversized Preta
```

## 11. Módulos do produto

### 11.1 Módulo de autenticação

Funções:

- Cadastro de lojista.
- Login.
- Recuperação de senha.
- Refresh token.
- Verificação de e-mail.
- Sessões ativas.
- Controle de dispositivos.

Regras:

- Senhas com hash forte.
- E-mail único.
- Login deve retornar usuário e loja ativa.
- Usuário pode ter uma ou mais lojas no futuro.
- No MVP, limitar uma loja por conta para simplificar.

### 11.2 Módulo de lojas

Campos principais:

- Nome da loja.
- Slug.
- Descrição.
- Segmento.
- CNPJ ou CPF opcional no MVP.
- WhatsApp.
- Instagram.
- Endereço.
- Horários.
- Logo.
- Banner.
- Status.
- Plano.

Regras:

- Slug deve ser único.
- Slug deve ser amigável.
- Slug não deve permitir palavras reservadas como `admin`, `api`, `app`, `login`, `www`.
- Loja pode estar em rascunho, publicada, suspensa ou arquivada.
- Loja suspensa não aparece publicamente.

### 11.3 Módulo de produtos

Campos:

- Nome.
- Descrição curta.
- Descrição completa.
- Categoria.
- Marca.
- SKU.
- Preço de venda.
- Preço promocional.
- Custo.
- Margem estimada.
- Estoque.
- Unidade.
- Peso.
- Dimensões.
- Fotos.
- Variações.
- Tags.
- Status.
- Destaque.

Status possíveis:

```text
DRAFT
ACTIVE
INACTIVE
OUT_OF_STOCK
ARCHIVED
```

Regras:

- Produto rascunho não aparece na vitrine.
- Produto inativo não aparece na vitrine.
- Produto sem preço pode aparecer como “consultar valor”, se permitido pela loja.
- Produto sem estoque pode aparecer como indisponível ou ser ocultado, conforme configuração.
- Produto pode ter preço “a partir de” para serviços ou itens sob orçamento.

### 11.4 Módulo de serviços

Nem todo lojista vende produto físico. O Vitrine deve tratar serviços como cidadão de primeira classe.

Campos:

- Nome do serviço.
- Descrição.
- Preço fixo ou “a partir de”.
- Duração estimada.
- Categoria.
- Fotos.
- Profissional responsável opcional.
- Observações.
- Botão de solicitar orçamento.

Exemplos:

- Corte masculino.
- Manicure.
- Design de sobrancelha.
- Instalação elétrica.
- Ensaio fotográfico.
- Consultoria.

### 11.5 Módulo de categorias

Campos:

- Nome.
- Descrição.
- Imagem.
- Ordem.
- Status.

Regras:

- Categoria pertence a uma loja.
- Produto pode pertencer a uma categoria principal.
- Futuramente produto pode ter múltiplas categorias.
- Categoria vazia pode ou não aparecer, conforme configuração.

### 11.6 Módulo de imagens

Requisitos:

- Upload simples.
- Compressão automática.
- Geração de thumbnails.
- Otimização para web.
- CDN.
- Limite por plano.
- Remoção segura.

Regras:

- Produto deve ter imagem principal.
- Ordem das imagens deve ser configurável.
- Imagens devem ser convertidas para formato otimizado quando possível.
- O sistema deve evitar upload gigante sem compressão.

### 11.7 Módulo de estoque simples

No MVP, estoque deve ser simples.

Configurações:

- Não controlar estoque.
- Controlar quantidade exata.
- Mostrar apenas disponibilidade.
- Mostrar “consulte disponibilidade”.

Regras:

- Se estoque for 0, produto pode ficar indisponível.
- Se pedido for via WhatsApp, o estoque não deve baixar automaticamente no MVP sem confirmação do lojista.
- Futuramente, pedido confirmado pode baixar estoque.

## 12. Módulo de IA

A IA é um diferencial forte do Vitrine, mas deve ser aplicada de forma prática. Ela não deve ser um enfeite. Deve reduzir trabalho do lojista e melhorar a qualidade do catálogo.

### 12.1 Objetivo da IA

A IA deve ajudar em quatro frentes:

1. Cadastro mais rápido.
2. Catálogo mais bonito e profissional.
3. Sugestões comerciais.
4. Análise simples de desempenho.

### 12.2 IA para cadastro de produtos

O lojista pode escrever:

```text
camiseta oversized preta algodão premium 79,90
```

A IA transforma em:

```text
Nome: Camiseta Oversized Preta
Descrição curta: Camiseta oversized em algodão premium, ideal para compor looks casuais com conforto e estilo.
Categoria: Camisetas
Preço: R$ 79,90
Tags: oversized, camiseta preta, algodão, casual
```

### 12.3 IA por foto

Fluxo futuro:

1. Lojista tira foto do produto.
2. IA sugere nome.
3. IA sugere categoria.
4. IA sugere descrição.
5. IA sugere tags.
6. Lojista revisa e publica.

A IA não deve publicar sozinha sem confirmação.

### 12.4 IA para descrição

Botão:

```text
Melhorar descrição
```

Opções:

- Mais profissional.
- Mais curta.
- Mais vendedora.
- Mais objetiva.
- Mais elegante.
- Mais popular.
- Com foco em benefícios.

### 12.5 IA para SEO

A IA sugere:

- Título SEO.
- Meta description.
- Palavras-chave.
- Texto de compartilhamento.
- Descrição para Instagram.
- Legenda para WhatsApp.

Exemplo:

```text
Título SEO: Vestido Midi Floral Feminino | Loja Bella
Meta description: Conheça o vestido midi floral feminino da Loja Bella. Veja fotos, preço, tamanhos disponíveis e fale pelo WhatsApp.
```

### 12.6 IA para campanha

O lojista informa:

```text
Quero fazer uma promoção de Dia das Mães com vestidos e bolsas.
```

A IA gera:

- Nome da campanha.
- Texto curto.
- Sugestão de desconto.
- Banner textual.
- Produtos sugeridos.
- Mensagem para WhatsApp.
- Legenda para Instagram.

### 12.7 IA para precificação e margem

A IA pode ajudar o lojista a entender se está vendendo com margem saudável.

Entrada:

```text
Custo: R$ 40,00
Preço: R$ 59,90
Taxas estimadas: 5%
Embalagem: R$ 2,00
```

Resposta:

```text
Margem bruta aproximada: R$ 14,90
Margem percentual aproximada: 24,87%
Atenção: se houver frete grátis ou desconto, sua margem pode cair bastante.
```

A IA deve evitar aconselhamento financeiro irresponsável. Ela deve explicar como chegou ao número e sugerir revisão.

### 12.8 IA para análise do catálogo

A IA pode analisar:

- Produtos sem foto.
- Produtos sem descrição.
- Produtos sem preço.
- Produtos com descrição muito fraca.
- Categorias bagunçadas.
- Produtos mais acessados sem conversão.
- Promoções vencidas.

Exemplo de insight:

```text
Seu catálogo tem 36 produtos ativos. 9 estão sem descrição completa e 4 não têm foto principal. Produtos com foto tendem a gerar mais confiança para o cliente. Priorize atualizar estes itens primeiro.
```

### 12.9 IA como copiloto do lojista

Tela de chat interno:

```text
Pergunte ao Vitrine:
- Quais produtos tiveram mais acessos esta semana?
- Quais produtos estão sem foto?
- Crie uma campanha para liquidar estoque parado.
- Gere uma descrição melhor para este produto.
- Me ajude a calcular o preço ideal deste item.
```

### 12.10 Limites éticos da IA

A IA não deve:

- Inventar disponibilidade.
- Alterar preço sem autorização.
- Publicar promoção sem confirmação.
- Prometer aumento de vendas.
- Fazer recomendações fiscais ou jurídicas definitivas.
- Emitir nota fiscal automaticamente sem dados corretos e confirmação.

A IA deve sempre apoiar decisão humana.

## 13. Módulo de custos e formação de preço

Esse é um diferencial importante, porque muitos pequenos lojistas vendem sem conhecer o custo real do produto.

### 13.1 Objetivo

Ajudar o lojista a entender se o preço de venda cobre custo, taxas e margem.

### 13.2 Campos de custo

Por produto:

- Custo de compra ou produção.
- Embalagem.
- Taxa de cartão estimada.
- Taxa de marketplace, se houver.
- Imposto estimado.
- Frete absorvido.
- Comissão.
- Custo operacional estimado.

### 13.3 Cálculos

O sistema pode calcular:

```text
Custo total = custo base + embalagem + taxas + frete absorvido + comissão + outros custos
Lucro bruto = preço de venda - custo total
Margem bruta = lucro bruto / preço de venda * 100
Markup = preço de venda / custo total
```

### 13.4 Alertas

- Produto com margem negativa.
- Produto com margem muito baixa.
- Promoção que zera lucro.
- Cupom que deixa venda no prejuízo.
- Preço menor que custo.
- Taxa de pagamento não considerada.

### 13.5 Simulador

O lojista pode simular:

```text
Se eu der 10% de desconto, ainda tenho lucro?
Se vender no cartão em 3x, quanto sobra?
Se incluir frete grátis, qual deve ser o preço mínimo?
```

### 13.6 Valor estratégico

Esse módulo transforma o Vitrine de simples catálogo para ferramenta de gestão leve.

## 16. Carrinho e pedido via WhatsApp

### 16.1 Conceito

No MVP, o carrinho não é checkout. É uma lista de interesse organizada.

### 16.2 Fluxo

1. Cliente adiciona produtos.
2. Cliente revisa lista.
3. Cliente informa nome opcional.
4. Cliente clica em enviar para WhatsApp.
5. Sistema gera mensagem.
6. WhatsApp abre com mensagem preenchida.
7. Cliente envia.
8. Lojista conversa e finaliza.

### 16.3 Benefícios

- Reduz mensagens desorganizadas.
- Evita cliente mandar prints soltos.
- Ajuda lojista a entender o pedido.
- Mantém fechamento humano.
- Evita complexidade de pagamento no MVP.

### 16.4 Mensagem configurável

O lojista pode escolher modelo:

```text
Modelo direto
Modelo formal
Modelo com endereço
Modelo com observação de entrega
Modelo para orçamento
```

### 16.5 Dados do pedido

Mesmo que o cliente envie pelo WhatsApp, o sistema pode salvar intenção:

- Loja.
- Itens.
- Total estimado.
- Data.
- Origem.
- Dispositivo.
- Status: enviado para WhatsApp.

Isso permite analytics.

## 22. Analytics

### 22.1 Métricas do lojista

- Acessos à vitrine.
- Produtos mais visualizados.
- Cliques no WhatsApp.
- Produtos adicionados ao carrinho.
- Pedidos enviados.
- Campanhas mais acessadas.
- Origem do tráfego.
- Horários de maior acesso.

### 22.2 Métricas úteis, não vaidosas

Evitar painel cheio de gráfico inútil.

Mostrar respostas:

- Quais produtos chamaram mais atenção?
- Quais produtos precisam de foto melhor?
- Qual campanha trouxe mais cliques?
- Quantas pessoas clicaram no WhatsApp?
- Quais categorias são mais vistas?

### 22.3 Analytics com IA

Exemplo:

```text
Seu produto “Vestido Midi Floral” teve 84 visualizações esta semana, mas apenas 2 cliques no WhatsApp. Tente revisar preço, foto principal ou descrição.
```

### 22.4 Privacidade

Analytics deve evitar rastreamento invasivo. Priorizar dados agregados.
