## 29. Requisitos funcionais

### RF001 — Cadastro de lojista

O sistema deve permitir que um lojista crie uma conta informando nome, e-mail, telefone e senha.

### RF002 — Login

O sistema deve permitir login com e-mail e senha.

### RF003 — Criação de loja

O sistema deve permitir criar uma loja com nome, slug, segmento e WhatsApp.

### RF004 — Edição de loja

O lojista deve poder editar descrição, logo, banner, cores, horário, endereço e redes sociais.

### RF005 — Publicação de loja

O lojista deve poder publicar ou despublicar a loja.

### RF006 — Cadastro de categorias

O lojista deve poder criar, editar, ordenar e remover categorias.

### RF007 — Cadastro de produtos

O lojista deve poder criar, editar, duplicar, ativar, desativar e arquivar produtos.

### RF008 — Upload de imagens

O lojista deve poder enviar imagens para produtos, logo e banners.

### RF009 — Cadastro de serviços

O lojista deve poder cadastrar serviços com nome, descrição, valor e CTA.

### RF010 — Vitrine pública

O cliente deve poder acessar a vitrine pública sem login.

### RF011 — Busca

O cliente deve poder buscar produtos e serviços.

### RF012 — Filtros

O cliente deve poder filtrar por categoria, preço e disponibilidade.

### RF013 — Carrinho/lista de interesse

O cliente deve poder adicionar itens a uma lista antes de enviar para WhatsApp.

### RF014 — Envio via WhatsApp

O sistema deve gerar mensagem formatada com os itens selecionados.

### RF015 — Registro de intenção

O sistema deve registrar quando um cliente clica para enviar pedido via WhatsApp.

### RF016 — Dashboard

O lojista deve visualizar resumo de acessos, cliques, produtos mais vistos e pedidos enviados.

### RF017 — Promoções

O lojista deve criar promoções por produto.

### RF018 — Cupons

O lojista deve criar cupons conforme plano.

### RF019 — Campanhas

O lojista deve criar campanhas agrupando produtos e promoções.

### RF020 — Templates

O lojista deve escolher um template visual.

### RF021 — Personalização

O lojista deve personalizar cores, logo e banners dentro de limites.

### RF022 — IA para descrição

O sistema deve oferecer geração ou melhoria de descrição de produto usando IA.

### RF023 — IA para campanha

O sistema deve sugerir textos e campanhas usando IA.

### RF024 — Controle de plano

O sistema deve limitar recursos de acordo com o plano.

### RF025 — Assinatura

O lojista deve conseguir contratar, cancelar ou alterar plano futuramente.

### RF026 — Admin interno

Administrador deve conseguir gerenciar lojas, usuários, planos e suporte.

### RF027 — Exportação

O lojista deve poder exportar produtos e pedidos em CSV futuramente.

### RF028 — Domínio próprio

Plano Pro deve permitir domínio próprio futuramente.

### RF029 — Pagamento online futuro

O sistema deve permitir checkout integrado em fase futura.

### RF030 — Nota fiscal futura

O sistema deve permitir integração fiscal em fase futura.

## 30. Requisitos não funcionais

### RNF001 — Performance

A vitrine pública deve carregar rapidamente, com meta de p95 abaixo de 200 ms para HTML/cache em cenários normais.

### RNF002 — Disponibilidade

Meta inicial: 99,5%.  
Meta madura: 99,9%.

### RNF003 — Escalabilidade

O sistema deve suportar crescimento horizontal no backend e uso de CDN para assets.

### RNF004 — Segurança

Dados privados de uma loja nunca podem ser acessados por outra.

### RNF005 — LGPD

O sistema deve seguir princípios de minimização, finalidade, transparência e segurança.

### RNF006 — Mobile first

Painel e vitrine devem funcionar muito bem em celular.

### RNF007 — Acessibilidade

Contraste, navegação por teclado e textos alternativos devem ser considerados.

### RNF008 — Observabilidade

Logs, métricas e tracing devem permitir diagnosticar problemas.

### RNF009 — Backup

Banco deve ter backup automático em produção.

### RNF010 — Manutenibilidade

Código deve ser modular, testável e documentado.

### RNF011 — Baixo custo

Arquitetura deve evitar custos altos no início.

### RNF012 — SEO

Vitrines e produtos devem ter metadados adequados.

## 31. Regras de negócio

### RB001 — Slug único

Cada loja deve ter slug único.

### RB002 — Slug imutável ou controlado

Alterar slug pode quebrar links. Permitir alteração apenas com aviso e redirecionamento.

### RB003 — Produto pertence a uma loja

Produto nunca existe fora de loja.

### RB004 — Categoria pertence a uma loja

Categoria nunca é compartilhada entre lojas no MVP.

### RB005 — Limite por plano

Cada plano define limite de produtos, imagens, campanhas e IA.

### RB006 — Loja suspensa

Loja suspensa não pode ser acessada publicamente.

### RB007 — Loja em rascunho

Loja em rascunho é visível apenas para o lojista.

### RB008 — Produto ativo

Somente produto ativo aparece na vitrine.

### RB009 — Produto arquivado

Produto arquivado não aparece em listagens, mas histórico de pedidos mantém referência.

### RB010 — Pedido via WhatsApp

Pedido via WhatsApp é intenção, não confirmação de compra.

### RB011 — Total estimado

O total mostrado é estimado até confirmação do lojista.

### RB012 — Preço sob consulta

Loja pode permitir produtos com preço sob consulta.

### RB013 — Cupom não pode gerar valor negativo

Desconto nunca pode deixar total menor que zero.

### RB014 — Promoção expirada

Promoção expirada deve ser removida automaticamente da vitrine.

### RB015 — Custo invisível ao cliente

Custo e margem nunca aparecem na vitrine pública.

### RB016 — IA exige revisão

Conteúdo gerado por IA deve ser revisado antes de publicar.
