## 35. Critérios de aceite

### Lançamento por texto

Dado que o usuário digita:

```text
gastei 50 no mercado
```

Quando confirma o lançamento, então o sistema deve criar uma despesa de R$ 50 na data atual, categoria Mercado e conta padrão.

### Cartão parcelado

Dado que o usuário registra compra de R$ 1.200 em 6x, o sistema deve criar 6 parcelas de R$ 200 nas faturas correspondentes.

### Dinheiro livre

Dado que o usuário tem saldo de R$ 1.000 e despesas previstas de R$ 700, o sistema deve exibir dinheiro livre de R$ 300, respeitando reservas protegidas e receitas previstas.

### MEI limite

Dado que o usuário registra receitas profissionais, o sistema deve somá-las no faturamento anual do MEI e mostrar percentual do limite usado.

## 46. Instruções para implementação

## 46.1 Comece pelo núcleo

Não comece por nota fiscal, Open Finance ou imposto de renda. Comece pela dor diária:

- Lançar rápido.
- Entender saldo real.
- Controlar cartão.
- Ver mês.
- Perguntar para IA.

## 46.2 Desenvolva com dados reais de teste

Crie cenários:

### Cenário pessoa física

- Salário R$ 3.500.
- Aluguel R$ 900.
- Cartão R$ 1.200.
- Mercado R$ 800.
- Meta reserva R$ 5.000.

### Cenário MEI

- Faturamento mensal variável.
- Clientes.
- DAS.
- Notas.
- Despesas de negócio.
- Mistura pessoal/profissional.

### Cenário casal

- Duas rendas.
- Contas compartilhadas.
- Meta casa.
- Cartão individual.

## 46.3 Não construir sem protótipo

Antes do código pesado:

- Criar fluxo no Figma.
- Testar com 5 pessoas.
- Ver onde travam.
- Ajustar linguagem.

## 46.4 Padrão técnico

- Usar Clean Architecture.
- Separar domínio financeiro.
- Criar testes para regras de saldo, cartão, parcelas e dinheiro livre.
- Não misturar UI com regra de negócio.
- Criar motor de cálculo isolado.
- Criar adapter para IA isolado.

- Money.
- Account.
- Transaction.
- Category.
- CreditCard.
- Invoice.
- Goal.
- Debt.
- Workspace.
- FinancialForecast.

## 46.6 Testes obrigatórios

- Receita aumenta saldo.
- Despesa reduz saldo.
- Transferência não altera patrimônio total.
- Compra no cartão entra na fatura.
- Pagamento da fatura reduz conta.
- Parcelas aparecem nos meses corretos.
- Dinheiro livre considera despesas previstas.
- Receita prevista não confirmada tem peso controlado.
- Transação empresarial não mistura com pessoal.

## 49. Exemplo de algoritmo: dinheiro livre

```pseudo
function calcularDinheiroLivre(workspace, dataFinal):
    saldoAtual = somarSaldosContasAtivas(workspace)
    receitasPrevistas = somarReceitasPrevistasConfirmaveis(workspace, hoje, dataFinal)
    despesasPrevistas = somarDespesasPrevistas(workspace, hoje, dataFinal)
    faturasCartao = somarFaturasAbertasEVencendo(workspace, hoje, dataFinal)
    reservasProtegidas = somarMetasProtegidas(workspace)

    return saldoAtual + receitasPrevistas - despesasPrevistas - faturasCartao - reservasProtegidas
```

### Observações

- Receitas incertas podem ter peso menor.
- Gastos variáveis podem ser estimados pela média.
- Reservas podem ser protegidas total ou parcialmente.

## 51. Exemplo de algoritmo: projeção MEI

```pseudo
function projetarFaturamentoMEI(receitasAno, mesAtual):
    totalAtual = soma(receitasAno)
    mediaMensal = totalAtual / mesAtual
    projecaoAnual = mediaMensal * 12
    percentualLimite = totalAtual / limiteAnual

    return {
        totalAtual,
        mediaMensal,
        projecaoAnual,
        percentualLimite
    }
```

## 55. Conclusão

O FinanceOS tem potencial porque não nasce apenas de uma oportunidade de mercado, mas de uma dor humana. Dinheiro mal organizado causa ansiedade, brigas, dívidas, atrasos, medo e estagnação. Um produto que realmente ajuda as pessoas a entender, organizar e planejar suas finanças pode ter impacto profundo.

O caminho correto é construir com excelência, mas também com humildade: ouvir usuários reais, resolver problemas reais, evitar complexidade desnecessária e manter a missão acima da vaidade tecnológica.

Se o produto for simples o suficiente para qualquer pessoa usar, completo o suficiente para crescer com ela e inteligente o suficiente para reduzir sua carga mental, ele poderá ser muito mais do que um app.

Ele poderá se tornar uma plataforma de transformação financeira para pessoas e pequenos negócios.
