## Frase central

> **O RoadMap Academy transforma "o que devo estudar para evoluir na carreira?" num mapa concreto — trilhas progressivas do Júnior ao Arquiteto de Soluções, ensinadas com projetos reais e nas duas linguagens que dominam o backend corporativo: Java e C#.**

---

## Resumo

Cursos de programação normalmente falham em duas frentes: **ordem** (jogam tecnologias
soltas sem dizer o que vem antes do quê) e **profundidade real** (param no "hello world"
e escondem como o software é feito no mercado). O RoadMap Academy nasceu para resolver
as duas.

A plataforma organiza a jornada de um engenheiro backend em **quatro trilhas** encadeadas,
somando **53 módulos** e mais de **600 aulas**, cada uma escrita com densidade alta,
métricas reais, exemplos de código nas duas linguagens e ganchos para a prática. Ao final,
quem percorre tudo não sabe apenas "programar" — sabe **desenhar sistemas** e justificar
decisões de arquitetura.

# 1. Para quem é

O aluno-alvo é o **desenvolvedor que quer construir uma base sólida do zero** e evoluir
de forma estruturada até o papel de **Arquiteto de Soluções**. O percurso pressupõe apenas
lógica básica de programação — o resto é construído módulo a módulo.

| Perfil | Onde começa | O que ganha |
| --- | --- | --- |
| Iniciante / júnior | Trilha Júnior (mód. 01) | Fundamentos reais: terminal, linguagem, dados, SQL, Git, redes, Docker. |
| Pleno buscando profundidade | Trilha Pleno/Sênior (mód. 11) | Design patterns, testes, performance, mensageria, cloud, observabilidade. |
| Sênior mirando arquitetura | Trilha Arq. Software (mód. 36) | DDD, sistemas distribuídos, padrões de integração, avaliação de arquitetura. |
| Arquiteto em formação | Trilha Arq. Soluções (mód. 45) | RNFs, cloud avançada, System Design, migração de legado, negócios, IA. |

::callout{type="note" title="Uma meta concreta por trás do mapa"}
O RoadMap foi desenhado com um objetivo real em vista: **base sólida → Arquiteto de
Soluções → oportunidades no mercado internacional.** Por isso o conteúdo é denso e
focado em fundamentos que não expiram, e não em modismos de framework.
::

# 2. O que o RoadMap Academy é

- Um **mapa de carreira** navegável, com ordem explícita entre módulos e trilhas.
- Um **curso backend dual**: todo conceito de aplicação vem em Java **e** C#.
- Uma plataforma de **teoria + prática**: aulas ligadas a projetos e exercícios.
- Um **produto de verdade**: em produção, com login, progresso salvo e conteúdo versionado.

## 2.1 O que o RoadMap Academy não é

- Não é um bootcamp de front-end nem um curso de uma única stack.
- Não é uma lista de vídeos soltos sem fio condutor.
- Não é teoria pura: cada trilha desemboca em projeto prático.
- Não promete atalho — promete **profundidade em ordem**.

# 3. Filosofia pedagógica

## 3.1 Dual Java / C# em tudo

A decisão de projeto mais marcante: o curso é **dedicado a Java E C#** (Spring Boot +
ASP.NET Core). O aluno deve sair sabendo programar nas duas. Na prática, **todo bloco de
código de aplicação em Java tem um par idiomático em C#**, exibidos em **abas** (Java é o
padrão) — mesmo exemplo, mesmo domínio, traduzido para as convenções de cada plataforma.

::callout{type="tip" title="Por que duas linguagens?"}
Java/Spring e C#/.NET são as duas stacks mais pedidas no backend corporativo. Aprender os
conceitos em ambas mostra o que é **essencial** (a ideia) e o que é **acidental** (a
sintaxe). Quem entende Repository, DTO, injeção de dependência ou CQRS nas duas reconhece
o padrão em qualquer linguagem.
::

Mapeamentos padronizados entre os dois mundos incluem: Spring MVC ↔ ASP.NET Core,
JPA/Hibernate ↔ EF Core, JUnit/Mockito ↔ xUnit/Moq/FluentAssertions, Resilience4j ↔ Polly,
Spring AI ↔ Microsoft.Extensions.AI / Semantic Kernel, `BigDecimal` ↔ `decimal`, records
para Value Objects e MediatR para CQRS/domain events. Configuração e infraestrutura (YAML,
Dockerfile, SQL, HCL, PromQL) não recebem par — independem de linguagem.

## 3.2 Teoria colada à prática

Cada aula tende a fechar com um gancho para aplicar o que foi visto:

- Um **CTA para o Playground** quando a aula tem um exercício de System Design ligado.
- **Chips "coloque em prática"** com os projetos da trilha na última aula de cada módulo.
- Links recíprocos entre exercícios e as aulas que os fundamentam.

## 3.3 Densidade alta, autor único

As aulas são escritas para **estudo autônomo**: cada uma traz números e métricas concretas,
seções de "como aparece em entrevista", checklists e armadilhas comuns — para que o aluno
consiga aprofundar sozinho, sem depender de um instrutor ao lado.

# 4. Como a jornada se conecta

O valor do RoadMap está no **encadeamento**. Bancos de dados aparecem no Júnior (SQL básico)
e voltam no Pleno (índices, N+1, otimização); testes começam na pirâmide de testes e
reaparecem em CI/CD; a mensageria do Pleno prepara o EDA e os sistemas distribuídos do
Arquiteto. Nada é ilha — cada assunto é revisitado com mais profundidade quando o aluno
já tem repertório para absorvê-lo.

::callout{type="note" title="Ao final da trilha"}
Quem percorre as quatro trilhas e os nove projetos sai enxergando o sistema inteiro — do
primeiro `if` ao desenho de uma arquitetura distribuída — e sabendo **justificar** cada
decisão, que é exatamente o que se espera de um arquiteto.
::
